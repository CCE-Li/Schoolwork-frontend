# ~~认证接口文档 (Authentication API)~~

## 基础信息
- **Base URL**: `/api/auth`
- 无需认证即可访问

## 接口列表

### 1. 用户登录
- **URL**: `POST /api/auth/login`
- **描述**: 用户登录接口
- **请求体**:
  ```json
  {
    "username": "testuser",
    "password": "password123"
  }
  ```
  - `username`: 用户名
  - `password`: 密码
- **响应**: 登录成功的用户信息

### 2. 用户注册
- **URL**: `POST /api/auth/regist`
- **描述**: 用户注册接口
- **请求体**:
  ```json
  {
    "username": "newuser",
    "password": "password123",
    "email": "user@example.com"
  }
  ```
  - `username`: 用户名
  - `password`: 密码
  - `email`: 电子邮箱
- **响应**: 注册成功的用户信息

## 公共响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {
    // 用户信息
  }
}
```

## 错误码说明
- 200: 请求成功
- 400: 请求参数错误
- 401: 认证失败（用户名或密码错误）
- 409: 用户已存在（注册时）
- 500: 服务器内部错误

## 注意事项
1. 密码以明文形式传输，建议在生产环境中使用 HTTPS 加密
2. 注册成功后会同时创建用户基本信息记录
3. 默认管理员账号：
   - 用户名: admin
   - 密码: admin123
4. 注册时请确保用户名的唯一性
5. 密码要求：
   - 长度至少8位
   - 包含大小写字母和数字
   - 建议包含特殊字符

## 安全建议
1. 建议实现密码强度检查
2. 建议添加验证码功能防止暴力破解
3. 建议实现登录失败次数限制
4. 建议实现会话超时机制
5. 建议记录登录日志以便审计
