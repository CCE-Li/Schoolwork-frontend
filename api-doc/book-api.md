# ~~图书接口文档 (Book API)~~

## 基础信息
- **Base URL**: `/api/books`
- 无需认证即可访问

## 接口列表

### 1. 获取图书列表
- **URL**: `GET /api/books`
- **描述**: 获取图书列表，支持多种条件筛选
- **请求参数**:
  - `bid` (可选): 图书ID，精确查询单本图书
  - `author` (可选): 作者名称，模糊匹配
  - `title` (可选): 图书标题，模糊匹配
  - `tags` (可选): 图书标签，多个标签用逗号分隔
- **响应**: 图书列表

### 2. 获取图书详情
- **URL**: `GET /api/books/{bid}`
- **描述**: 获取图书详情页面
- **路径参数**:
  - `bid`: 图书ID
- **响应**: 图书详情页内容

### 3. 添加图书评论
- **URL**: `POST /api/books/{bid}`
- **描述**: 为指定图书添加评论
- **路径参数**:
  - `bid`: 图书ID
- **请求参数**:
  - `uid`: 用户ID
- **请求体**: 
  ```json
  {
    "rating": 5,
    "comment": "这本书非常棒！"
  }
  ```
  - `rating`: 评分 (1-5)
  - `content`: 评论内容
- **响应**: 操作结果

## 公共响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {} // 具体数据
}
```

## 错误码说明
- 200: 请求成功
- 400: 请求参数错误
- 403: 禁止访问
- 404: 资源不存在
- 500: 服务器内部错误

## 注意事项
1. 添加评论时，每个用户对同一本书只能评论一次
2. 评分必须为1-5之间的整数
3. 图书列表接口支持分页，但当前实现中分页参数未在前端使用，默认为获取所有匹配记录
