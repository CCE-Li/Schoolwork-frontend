# ~~购物车接口文档 (Cart API)~~

## 基础信息
- **Base URL**: `/api/cart`
- 需要用户登录才能访问

## 接口列表

### 1. 添加商品到购物车
- **URL**: `POST /api/cart/add`
- **描述**: 将商品添加到用户的购物车，如果商品已存在则增加数量
- **请求参数**:
  - `uid`: 用户ID (必须)
- **请求体**:
  ```json
  {
    "bid": 1,
    "number": 1
  }
  ```
  - `bid`: 图书ID
  - `number`: 添加数量
- **响应**: 操作结果

### 2. 获取购物车列表
- **URL**: `GET /api/cart/list`
- **描述**: 获取指定用户的购物车商品列表
- **请求参数**:
  - `uid`: 用户ID (必须)
- **响应**: 购物车商品列表

### 3. 删除购物车项
- **URL**: `POST /api/cart/delete`
- **描述**: 批量删除购物车中的商品
- **请求参数**:
  - `uid`: 用户UID (必须)
- **请求体**: 图书BID数组
  ```json
  [1, 2, 3]
  ```
- **响应**: 操作结果

### 4. 更新购物车商品数量
- **URL**: `POST /api/cart/update`
- **描述**: 修改购物车中某商品的数量
- **请求体**:
  ```json
  {
    "id": 1,
    "number": 2
  }
  ```
  - `id`: 购物车项ID
  - `number`: 新的商品数量
- **响应**: 操作结果

## 公共响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {} // 具体数据
}
```

## 错误码说明
- 200: 请求成功
- 400: 请求参数错误
- 401: 未授权/未登录
- 403: 禁止访问
- 404: 资源不存在
- 500: 服务器内部错误

## 注意事项
1. 所有接口都需要提供有效的 `uid` 参数
2. 添加商品时，如果购物车中已存在相同商品，则自动累加数量
3. 更新数量时，请确保提供正确的购物车项ID和新的数量值
4. 删除操作支持批量删除，需要提供要删除的图书ID数组
